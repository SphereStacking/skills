# Go/No-Go チェックリストパターン

各フェーズに関連する項目を選択する。全ての項目が全フェーズに適用されるわけではない。

## 必須 vs 推奨の判断基準

- **✅ 必須**: 不合格のまま進むと**データ損失・セキュリティ露出・本番障害・ロールバック不能**のリスクがある項目
- **⚠️ 推奨**: 不合格でも進行可能だが、**技術的負債・品質低下・開発体験の悪化**につながる可能性がある項目

迷ったら ✅ 必須にする。推奨に下げるのは「失敗しても後から対処可能」な場合のみ。

| 項目 | 検証方法 | 必須 |

## API / バックエンド

| 項目 | 検証方法 | 必須 |
|------|---------|------|
| 全エンドポイント 200 返却 | 各エンドポイントに `curl` | ✅ |
| 認証エンドポイント 401 返却 | 認証トークンなしで `curl` | ✅ |
| パブリックレスポンスに機密データなし | レスポンスの手動確認 | ✅ |
| レスポンス形状がフロントエンドと一致 | useFetch/fetch 呼び出しと比較 | ✅ |
| スモークテスト通過 | 新規 API テストの `{テストコマンド}` | ✅ |
