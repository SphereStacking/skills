# プロジェクトオーケストレーター

`/orchestrate` コマンドで開始。2つのモード:

- **GENERATE**: `/orchestrate generate` — 合意済みの計画をフェーズドキュメントに構造化
- **EXECUTE**: `/orchestrate execute phase 0` or `/orchestrate continue` — フェーズを実行

計画立案（探索・質問・フェーズ分割の合意）は Claude の Plan モードで行い、計画が固まったら `/orchestrate generate` でフェーズドキュメントに変換する。

## 主要コンセプト

- **マネージャーパターン**: メインセッションは構造化・委譲のみ。絶対に自分で実装しない
- **Plan モードとの連携**: 計画は Claude 標準の Plan モードに任せ、Orchestrator は永続化・委譲・ゲート管理に特化
- **実装者エージェント**: `project-orchestrator:implementer` — 動的プロンプトで任意の技術スタックに適応
- **ペルソナ**: 堅実型/防御型/適応型/検証型 — フェーズの性質に応じてマネージャーが選択・注入
- **レビューサイクル**: 3エージェント並列（コードレビュー、専門家、批判的）
- **トリアージポリシー**: Critical/High → 修正、Medium → 評価、Low → 破棄
- **Go/No-Go ゲート**: 次フェーズへ進む前にユーザー承認必須
- **複数プラン管理**: `.claude/works/{NNN}-{slug}/` 配下にプランごとに独立管理
- **セッション継続**: ディスク上のフェーズドキュメントがセッションをまたいで永続化
- **バリデーション**: `validate_phase.py` スクリプトでフェーズドキュメントの構造を自動検証
