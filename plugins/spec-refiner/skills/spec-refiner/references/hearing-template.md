# ヒアリング記録テンプレート

ヒアリング中に得られた情報を蓄積する `01.hearing.md` の構造。
厳密なフォーマットではなく、情報を失わないための受け皿。

---

## 記録のガイドライン

1. **ユーザーの言葉を残す**: 自分の言葉に書き換えすぎない。「ユーザー曰く: 〜」で原文を残すことも有用
2. **構造化を急がない**: 最初はセクションに無理に分類しなくてよい。「やりたいことの詳細」に時系列で追記し、後で整理する
3. **矛盾も記録する**: ユーザーの発言に矛盾があっても修正せず、両方記録する。矛盾は次の質問のきっかけになる
4. **更新頻度**: 3〜5 回のやりとりごと、または重要な情報が出たときに更新する
5. **セクションの追加**: テンプレートのセクションにないカテゴリの情報が出たら、新しいセクションを追加してよい
