# ヒアリングワークフロー

spec-refiner の対話の進め方と、記録の更新プロセス。

---

## 対話サイクル

```
ユーザーの発言 → 言語化して確認 → 自然に次の質問 → hearing.md に蓄積
```

これを**ユーザーが終了を告げるまで**繰り返す。

---

## 対話のフロー

### 1. 受け取る

ユーザーの発言を注意深く聞く。以下の情報を拾う:

- **事実**: 具体的な要件、仕様、制約
- **意図**: なぜそうしたいのか、背景にある目的
- **感情**: こだわり、不安、期待、苛立ち
- **言外の意味**: 明示されていないが暗黙に前提としていること

### 2. 言語化する

ユーザーの発言を自分の言葉で言い換え、確認する。

**良い例:**
- 「つまり、管理者だけが設定を変更できて、一般ユーザーは閲覧のみ、ということですね」
- 「今の話をまとめると、まず X があって、それをトリガーに Y が起こる、という流れですか」

**悪い例:**
- 「承知しました」（言語化していない）
- 「つまり RBAC ベースのアクセス制御が必要ですね」（ユーザーの言葉でなく専門用語に置き換えている）

### 3. 記録する

`hearing.md` の適切なセクションに情報を追記する。

- 新しいトピックが出たら「やりたいことの詳細」にサブセクションを追加
- 既存のトピックの深掘りならそのサブセクションに追記
- どこにも当てはまらなければ「メモ」に入れて後で整理

### 4. 次の質問を選ぶ

**最優先: ユーザーの直前の発言から自然に派生する質問**

ユーザーが「通知機能が欲しい」と言ったら:
- 「どんなタイミングで通知が届くといいですか？」 ← 自然な派生
- ~~「データモデルはどうしますか？」~~ ← 脈絡がない

**次点: 気になっているが聞けていないこと**

対話が一区切りついた（ユーザーが「そんな感じかな」等と言った）場合:
- `hearing-guide.md` の質問レパートリーから、まだ触れていない切り口の質問を選ぶ
- ただし「次はこのテーマについて聞きます」とは言わない。自然に話題を振る

---

## サマリーのタイミング

### 10 回以上のやりとりが経過したら

一度立ち止まって、ここまでの内容を整理して提示する:

「ここまでの話を一度整理させてください:

- {要点 1}
- {要点 2}
- {要点 3}
...

この理解で合っていますか？抜けている部分や、違う部分があれば教えてください。」

これにより:
- ユーザーが自分の発言を俯瞰できる
- 認識のズレを早期に発見できる
- ユーザーが「あ、そういえばこれも」と新しい情報を出すきっかけになる

### サマリー後

ユーザーの確認/修正を反映し、ヒアリングを続行する。

---

## コードベース分析の活用

`spec-researcher` エージェントのバックグラウンド分析結果は、以下の場面で活用する:

1. **ユーザーが既存システムに言及した場合**: 「コードベースを見ると {X} がありますが、これと関係しますか？」
2. **技術的な質問が出た場合**: 「現在のプロジェクトでは {Y} を使っていますが、それを前提にしますか？」
3. **制約の確認**: 「既存の {Z} との整合性は意識しますか？」

分析結果を**こちらから一方的に提示しない**。ユーザーの話の文脈で自然に組み込む。

---

## 終了時の処理

ユーザーが終了を告げたら:

### 1. hearing.md の最終更新

全ての未記録情報を `hearing.md` に反映する。

### 2. 全体サマリーの提示

ヒアリング内容の全体サマリーを構造化して提示する:

```
## ヒアリングサマリー

### やりたいこと
{1〜3 文の概要}

### 主なポイント
- {ポイント 1}
- {ポイント 2}
- ...

### まだ決まっていないこと
- {未決事項 1}
- {未決事項 2}

### 保存先
`.claude/spec-refiner/{NNN}-{slug}/hearing.md`
```

### 3. 次のステップ案内

「ヒアリング内容は `.claude/spec-refiner/{NNN}-{slug}/hearing.md` に保存しました。

次のステップ:
- `/spec-architect review .claude/spec-refiner/{NNN}-{slug}/hearing.md` で仕様の深掘りとレビューに進めます
- `/spec-refiner continue` でヒアリングを再開することもできます」
